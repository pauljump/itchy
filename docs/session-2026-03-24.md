# Itchy Session Log — 2026-03-24

## Summary
Built first byte-level submission for OpenAI Parameter Golf competition in one session.
Final architecture: 17.5M param byte-level transformer, patch=12, per-position decode heads.
Best validation BPB: 0.16-0.17 across 2, 10, and 40 shard runs. Competition SOTA is 1.12.

### Phase 2: Built Itchy v1 — Byte-Level Model
- Created `model_itchy.py`: byte-level transformer with LoRA adapters
  - 256 vocab (raw bytes), no tokenizer
  - Patch processing: group 4 bytes → 1 patch
  - LoRA adapters in every block (zero-initialized, for TTT)
  - 17.3M params, 13.0MB at int6
- Created `train_itchy_mlx.py`: MLX training loop
- Created `data/convert_to_bytes.py`: converts sp1024 tokenized shards to raw byte shards
- Verified forward pass and loss on synthetic data
- Ran 15 steps locally: loss dropped 5.8 → 1.5 nats. Model learns.

### Phase 3: TTT Meta-Learning (built + failed)
- Built TTT meta-learning: 20% of training steps are adapt-then-predict episodes
- Tested on Mac: adapter TTT showed zero improvement
- Tested on T4 Colab: trained 3000 steps with meta-learning. Results:
  - All-param TTT: helped 100% of docs (delta -0.024)
  - Adapter TTT: helped 0% of docs (delta 0.000)
- **Conclusion**: LoRA adapter mechanism is dead. Zero-init gate creates gradient dead zone.
- Stripped adapters entirely. All params → model capacity.

### Phase 4: Applied Competition Tricks (mixed results)
- Ablation study on free T4, 7 configs, 3000 steps each:

- **Reject**: Partial RoPE, LN Scale, N-gram hashing (all hurt or no effect at byte level)

### Phase 6: Patch Size Discovery (THE breakthrough)
- Ablation on free T4, patch sizes 2-16:

| Patch | Val BPB | Delta vs 4 |
|-------|---------|------------|
| 2 | 1.1512 | +0.557 |
| 3 | 0.7800 | +0.186 |
| 4 | 0.5944 | — |
| 6 | 0.4119 | -0.183 |
| 8 | 0.3298 | -0.265 |
| **12** | **0.2903** | **-0.304** |
| 16 | 0.3343 | -0.260 |

- **Patch size 12 is the dominant finding.** -0.304 BPB improvement.
- This is 40x bigger than all competition tricks combined.
- Optimal model search confirmed: local encoder (MegaByte-style) adds nothing at this scale.

### Phase 7: Per-Position Decode Heads (second breakthrough)
- The unpatch layer (patch repr → 12 byte predictions) was a bottleneck
- Tested 5 decode strategies:

| Decode | Val BPB | Delta |
|--------|---------|-------|
| Flat linear (baseline) | 0.2903 | — |
| +Pos embed + shared head | 3.2600 | broken |
| **+Per-pos MLP heads** | **0.2329** | **-0.057** |
| Autoregressive decoder | 0.7937 | too complex |
| MoE unpatch | 3.2594 | broken |

- Each byte position gets its own small MLP (dim → 128 → 260)
- Second biggest finding after patch size.

## Final Architecture

```
Input: raw UTF-8 bytes (0-255)
  → nn.Embedding(260, 384)
  → Reshape: group 12 bytes → 1 patch
  → nn.Linear(384*12, 384): patch projection
  → RMSNorm
  → 11x Block:
      - Residual mix with x0
      - CausalSelfAttention (8 heads, 4 KV, full RoPE)
      - LeakyReLU(0.5)² MLP (3x expansion)
      - Encoder-decoder skip connections
  → RMSNorm
  → PerPositionDecode: 12 independent MLP heads (384 → 128 → 260)
  → Softcap(30)
Output: 12 byte predictions per patch

17,480,152 params | ~13.1MB at int6 | patch=12
```

## Key Decisions and Why

1. **Byte-level over token-level**: 256 vocab means tiny embedding table. All params go to transformer. At matched size, byte model has 4x more transformer capacity.

2. **Patch size 12**: Sweet spot. 12 bytes ≈ 2 English words. Smaller patches waste attention on too many positions. Larger patches (16) lose too much byte-level detail. This single decision was worth -0.304 BPB.

3. **Per-position decode**: A flat linear projection forces all 12 byte predictions to be independent. Giving each position its own MLP lets the model specialize. Worth -0.057 BPB.

4. **LeakyReLU(0.5)² + 3x MLP**: Only competition tricks that transferred to byte level. Everything else (partial RoPE, LN scaling, n-gram hashing) hurt or had no effect.

5. **No adapters, no TTT meta-learning**: Tested thoroughly, doesn't work. Zero-init gate creates gradient dead zone. Stripped to free params for capacity.

6. **EMA (0.997 decay)**: Consistent -0.004 to -0.005 BPB improvement for free.

## What Didn't Work (with evidence)
- LoRA adapters: 0% improvement across 3 experiments
- TTT meta-learning: adapters can't learn to adapt in 3 SGD steps
- Partial RoPE: +0.011 BPB (hurts)
- LN Scale per layer: +0.012 BPB (hurts)
- Hash n-gram embeddings: +0.000 BPB (no effect)
- Autoregressive local decoder: +0.503 BPB (too complex for scale)
- MoE unpatch: +2.969 BPB (broken)
- Local/global encoder: +0.000 BPB (no effect)
